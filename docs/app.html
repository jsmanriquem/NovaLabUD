app module#

class app.DataOperationsWithUI(ui_container)#

Bases: DataOperations

Extensión de DataOperations que integra callbacks para la interfaz de usuario.

Esta clase extiende las operaciones de datos básicas para actualizar automáticamente la interfaz de usuario cuando se realizan operaciones.

ui_container#

Referencia al contenedor de UI principal.

Inherits:

DataOperations: Clase base con operaciones de datos fundamentales.

fill_null_with_mean(ui_callback=None)#

Rellena los valores nulos en columnas numéricas con la media de cada columna y actualiza la UI si se proporciona un callback.

Args: ui_callback (callable, optional): Función que se llama para actualizar la UI con los datos modificados, si se proporciona.

load_file(ui_callback=None)#

Carga un archivo y actualiza la UI si es exitoso.

Parámetros:

ui_callback (callable, optional) – Función a llamar para actualizar la UI con los datos cargados.

Devuelve:

True si la carga fue exitosa, False en caso contrario

Tipo del valor devuelto:

bool

normalize_data(ui_callback=None)#

Normaliza las columnas numéricas del conjunto de datos y actualiza la UI si se proporciona un callback.

Args: ui_callback (callable, optional): Función que se llama para actualizar la UI con los datos modificados, si se proporciona.

remove_duplicates(ui_callback=None)#

Elimina las filas duplicadas del conjunto de datos y actualiza la UI si se proporciona un callback.

Args: ui_callback (callable, optional): Función que se llama para actualizar la UI con los datos modificados, si se proporciona.

remove_null_values(ui_callback=None)#

Elimina los valores nulos del conjunto de datos y actualiza la UI si se proporciona un callback.

Args: ui_callback (callable, optional): Función que se llama para actualizar la UI con los datos modificados, si se proporciona.

class app.LaboratorySoftware#

Bases: object

Clase principal del Software de Laboratorio para análisis y procesamiento de datos. Esta clase implementa la interfaz gráfica principal y coordina todas las funcionalidades del software, incluyendo la carga de datos, visualización, procesamiento y análisis.

root#

Ventana principal de la aplicación.

Type:

tk.Tk

main_frame#

Panel principal dividido que contiene la tabla de datos y resultados.

Type:

ttk.PanedWindow

data_frame#

Marco para la tabla de datos.

Type:

ttk.LabelFrame

results_frame#

Marco para mostrar resultados y gráficas.

Type:

ttk.LabelFrame

data_table#

Tabla para visualizar los datos cargados.

Type:

ttk.Treeview

data_ops#

Instancia para operaciones de datos con UI.

Type:

DataOperationsWithUI

no_data_label#

Etiqueta mostrada cuando no hay datos cargados.

Type:

ttk.Label

Clase principal del Software de Laboratorio.

create_data_table()#

Inicializa un Treeview con scrollbars vertical y horizontal para mostrar los datos cargados de manera organizada y navegable. Crea la tabla para mostrar los datos con scrollbars.

run() → None#

Inicia el bucle principal de la aplicación.

Este método debe ser llamado después de la inicialización para comenzar la ejecución de la interfaz gráfica.

setup_menus() → None#

Configura la barra de menús de la aplicación.

Crea y configura los menús principales: - Archivo: Para operaciones de importación/exportación - Edición: Para procesamiento de datos - Acerca de: Para información y documentación

show_autores()#

Muestra los autores del software.

update_data_display(data: DataFrame)#

Actualiza la tabla de datos con nueva información.

Parámetros:

data (pd.DataFrame) – DataFrame con los nuevos datos a mostrar. Si es None o está vacío, se muestra el mensaje de no datos.
